Remix JS vs Next JS

By George Holmes | 3/17/25

Remix JS is a relatively new JavaScript framework created in 2020. Recently, it has been taking off with websites like ChatGPT (OpenAI), Docker, Shopify, and NASA using Remix JS. But is it really that good compared to Next JS?

For about a month now, I have been using Next JS in all of my projects, but that was before I learned about Remix JS. But before I tell you which one is better, let's first compare them.

Feature Remix JS Next JS
TypeScript support Yes Yes
Nested routes Yes Yes
Server side rendering Yes Yes
Pre-fetching Yes No
Pending UI Yes Yes
Dynamic file picks No Yes
Authentication Yes (kinda) Yes

Chart by George Holmes

To be fair, comparing these frameworks together is not really fair, but they are very similar. They both have the most important features like dynamic server-side rendering and pending UI. However, in terms of how supported they are by something like Supabase (one of my favorites), Next JS wins.

Now in other areas like UI components, I think that Remix JS wins, especially for its great pre-rendering UI components with React and Redux for user state management.

In conclusion, generally for big projects, I would say use Next.js mainly for its built-in tools and great MVC. However, I would still strongly suggest trying out Remix JS in a small project.
